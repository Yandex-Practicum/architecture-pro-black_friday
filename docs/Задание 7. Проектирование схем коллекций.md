# Задание 7. Проектирование схем коллекций

## 1. Коллекция orders

**Атрибуты:**`order_id`, `customer_id`, `created_at`, `items[]`, `status`, `total_amount`, `geo_zone`

**Основные операции:**

* Создание заказов (высокая частота в Black Friday)
* Поиск истории заказов по `customer_id`
* Отображение статуса заказа

**Анализ кандидатов:**


| Поле             | Кардинальность | Равномерность | Проблемы                                         |
| -------------------- | ---------------------------- | -------------------------- | -------------------------------------------------------- |
| `order_id`(ObjectId) | Высокая               | Неравномерная | Монотонный рост → горячий шард |
| `customer_id`        | Высокая               | Равномерная     | Хороший кандидат                          |
| `geo_zone`           | Низкая                 | Неравномерная | Мало значений → перекос              |
| `created_at`         | Высокая               | Неравномерная | Монотонный → горячий шард          |

Выбор: `{ customer_id: "hashed" }` — Hashed шардирование

История заказов всегда ищется по `customer_id`, при Hashed все заказы одного клиента на одном шарде (scatter-gather минимален), равномерное распределение

## 2. Коллекция products

**Атрибуты:**`product_id`, `name`, `category`, `price`, `stock_by_geo{}`, `attributes{}`

**Основные операции:**

* Частые обновления остатков (при покупках)
* Поиск по категориям + фильтрация по цене
* Страница продукта (поиск по `product_id`)

**Анализ кандидатов:**


| Поле                   | Кардинальность | Равномерность | Проблемы                                   |
| -------------------------- | ---------------------------- | -------------------------- | -------------------------------------------------- |
| `product_id`               | Высокая               | Равномерная     | Хороший кандидат                    |
| `category`                 | Низкая                 | Неравномерная | "Электроника" = горячий шард |
| `{ category, product_id }` | Высокая               | Средняя             | Составной — лучше чем category   |

Выбор:`{ product_id: "hashed" }` — Hashed шардирование

Обновления остатков идут точечно по `product_id`  — попадают в конкретный шард. Category как ключ создает горячие шарды

## 3. Коллекция carts

**Атрибуты:**`_id`, `user_id`, `session_id`, `items[]`, `status`, `created_at`, `updated_at`, `expires_at`

**Основные операции:**

* Поиск активной корзины по `{ session_id, status }` или `{ user_id, status }`
* Добавление и удаление товаров
* Слияние гостевой корзины в пользовательскую

**Анализ кандидатов:**


| Поле      | Кардинальность | Равномерность | Проблемы                                      |
| ------------- | ---------------------------- | -------------------------- | ----------------------------------------------------- |
| `user_id`     | Высокая               | Равномерная     | Авторизованные пользователи |
| `session_id`  | Высокая               | Равномерная     | Гости, но нет связи с user\_id        |
| `_id`(hashed) | Высокая               | Равномерная     | Scatter-gather на поиск корзины         |

Выбор: составной ключ `{ user_id: "hashed" }` с дополнительным индексом на `session_id`

Основные операции фильтруют по `user_id`. Гостевые корзины хранятся с `session_id`, но после слияния становятся корзинами пользователя. TTL через `expires_at` автоматически чистит старые корзины.

## Команды MongoDB

```
// Подключаемся к mongos
use somedb

// === ORDERS ===
sh.enableSharding("somedb")

db.orders.createIndex({ customer_id: "hashed" })

sh.shardCollection("somedb.orders", { customer_id: "hashed" })

// === PRODUCTS ===
db.products.createIndex({ product_id: "hashed" })

sh.shardCollection("somedb.products", { product_id: "hashed" })

// Индекс для поиска по категории + цене
db.products.createIndex({ category: 1, price: 1 })

// === CARTS ===
db.carts.createIndex({ user_id: "hashed" })

sh.shardCollection("somedb.carts", { user_id: "hashed" })

// TTL индекс для автоудаления
db.carts.createIndex({ expires_at: 1 }, { expireAfterSeconds: 0 })

// Индекс для поиска гостевых корзин
db.carts.createIndex({ session_id: 1, status: 1 })
```

## Риски выбранных ключей

orders / customer_id hashed → scatter-gather при запросах без customer_id (поиск по статусу)
products / product_id hashed → запросы по категории всегда идут на все шарды
carts / user_id hashed → гостевые корзины (session_id) тоже scatter-gather

## Итоговая таблица решений

| Коллекция | Шард-ключ | Стратегия | Обоснование                                                                                     |
| ------------------ | ----------------- | ------------------ | ---------------------------------------------------------------------------------------------------------- |
| orders             | `customer_id`     | Hashed             | Равномерное распределение, запросы по клиенту в 1 шард        |
| products           | `product_id`      | Hashed             | Точечные обновления остатков, избегаем горячей категории |
| carts              | `user_id`         | Hashed             | Основные операции по пользователю, TTL через expires\_at                |
