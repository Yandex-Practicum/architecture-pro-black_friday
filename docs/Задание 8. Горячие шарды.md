## Задание 8. Горячие шарды

### 8.1 Метрики мониторинга


| Метрика                          | Инструмент      | Порог сработки     | Что случилось, если сработал порог            |
| --------------------------------------- | ------------------------- | ------------------------------- | -------------------------------------------------------------------------- |
| Кол-во чанков на шард  | `config.chunks`           | Разброс > 20%            | Неравномерное распределение данных         |
| Размер данных на шард | `getShardDistribution()`  | Разброс > 30%            | Один шард перегружен данными                      |
| CPU узла шарда                 | `mongostat`/ Prometheus   | > 80% sustained                 | Вычислительная перегрузка                          |
| Операций/сек на шард   | `serverStatus.opcounters` | В 2x выше среднего | Горячий шард по нагрузке                              |
| Очередь блокировок     | `globalLock.currentQueue` | > 10                            | Запросы встают в очередь                              |
| Latency чтения                    | `opLatencies.reads`       | > 100ms                         | Деградация производительности                  |
| Jumbo-чанки                        | `config.chunks`           | > 0                             | Балансировщик не может переместить чанк |

javascript

```javascript
// Распределение чанков по шардам
use config
db.chunks.aggregate([
  { $match: { ns: "somedb.products" } },
  { $group: { _id: "$shard", count: { $sum: 1 } } },
  { $sort: { count: -1 } }
])

// Размер данных на каждом шарде
use somedb
db.products.getShardDistribution()

// Поиск jumbo-чанков
use config
db.chunks.find({ ns: "somedb.products", jumbo: true })

// Нагрузка в реальном времени (выполнять на каждом mongod)
db.serverStatus().opcounters
db.serverStatus().opLatencies
db.serverStatus().globalLock.currentQueue

// Активные долгие запросы
db.adminCommand({ currentOp: 1, active: true, secs_running: { $gt: 1 } })
```

#### 8.2 Меры устранения дисбаланса

##### Балансировщик MongoDB

```javascript
// Статус балансировщика
sh.getBalancerState()
db.adminCommand({ balancerStatus: 1 })

// Включить / выключить
sh.startBalancer()
sh.stopBalancer()

// История перемещений чанков
use config
db.actionlog.find({ what: "moveChunk.from" }).sort({ time: -1 }).limit(10)
```

**Пороги автоматической балансировки:**


| Кол-во чанков на шарде | Порог разницы для балансировки |
| ---------------------------------------- | --------------------------------------------------------- |
| < 20                                     | 2 чанка                                              |
| 20 – 79                                 | 4 чанка                                              |
| ≥ 80                                    | 8 чанков                                            |

##### Окно балансировки

```javascript
// Ограничить балансировку ночными часами (время в UTC)
use config
db.settings.updateOne(
  { _id: "balancer" },
  { $set: { activeWindow: { start: "23:00", stop: "03:00" } } },
  { upsert: true }
)

// Убрать ограничение
db.settings.updateOne(
  { _id: "balancer" },
  { $unset: { activeWindow: 1 } }
)
```

##### Ручное перераспределение jumbo-чанков

```javascript
// Разбить jumbo-чанк
sh.splitAt("somedb.products", { product_id: <split_value> })

// Переместить чанк на другой шард
sh.moveChunk(
  "somedb.products",
  { product_id: <value_in_chunk> },
  "shard2"
)
```
