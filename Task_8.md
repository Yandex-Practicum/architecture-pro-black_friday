# Задача 8

## Проблема
Из-за категории «Электроника» произошла перегрузка одного из шардов MongoDB, так как 70% запросов приходилось именно на эти товары. Поэтому сейчас нужно разработать стратегию, как выявлять и устранять такие «горячие» шарды, а ещё предложить метрики мониторинга, чтобы в будущем можно было предотвращать такие ситуации. Не забудьте учесть, что товары из популярных категорий могут создавать непропорциональную нагрузку на отдельные узлы.


## Решение

### Определение "горячих" шардов
   
Введение монитринга. Частота сбора 60 секунд.

Для определения "горячих" шардов нужно отслеживать ряд ключевых метрик для мониторинга:
- время отклика выполнения запроса
- количество запросов/ответо в секунду
- использование памяти
- количество одновременных подключений к шарду

Нужно сгруппировать метрики по тому, с какой целью запрашиваются данные из БД. 

Необходимо провести анализ запросов, которые создают ситуацию с "горячими" шардами.

### Стратегия устранения перегрузок

#### Краткосрочные: 
- вертикальное масштабирование сервера с "горячим" шардом
     - добавить реплик
     - проверить ресурс жесткого диска.  
- кеширование на уровне приложения 
- ограничение количества запросов в пиковые моменты

#### Долгосрочные
- Смена шарда с ключа category на составной ключ category/price
```javascript
   sh.addShardTag("shard01", "electronics")
   sh.addShardTag("shard02", "electronics")
   sh.addTagRange(
      "db.products",
      {category: "Электроника", product_id: MinKey},
      {category: "Электроника", product_id: MaxKey},
      "electronics"
   )
```
