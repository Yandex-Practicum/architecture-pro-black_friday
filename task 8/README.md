### 1. Mетрики мониторинга

Для своевременного выявления перегрузки шардов необходимо отслеживать следующие метрики (с интервалом сбора 1–5 мин):

**Системные метрики шарда:**
- загрузка CPU на хостах шарда;
- используемая оперативная память;
- операции ввода‑вывода в секунду;
- входящий/исходящий трафик.

**Метрики производительности MongoDB:**
- количество операций чтения (запросов/сек);
- количество обновлений (операций/сек);
- количество вставок(операций/сек);
- число активных клиентских соединений;
- свободные соединения;
- количество блокировок на чтение/запись.

**Метрики шардирования:**
- число чанков в шарде;
- средний размер чанка;
- текущие миграции;
- время последней миграции (мс);

**Прикладные метрики по категориям:**
- частота запросов по категориям (запросов/час);
- частота обновлений остатков (обновлений/час);
- распределение запросов по значениям шард‑ключа.

**Алерты (триггеры для действий):**
- CPU > 85% в течение 5 мин;
- Память > 90% в течение 10 мин;
- IOPS > 90% от лимита;
- Среднее время ответа > 500 мс в течение 2 мин;
- Дисбаланс чанков (> 2× от среднего по кластеру).

### 2. Механизмы автоматического перераспределения

#### 2.1. Динамическая решардировка (на основе нагрузки)

**Алгоритм:**
1. Каждые 15 мин анализируется метрика "частота запросов по категориям (запросов/час)".
2. Если для категории нагрузка > 60% от общей за последние 30 мин:
   - Разделить диапазон значений шард‑ключа для категории на поддиапазоны.
   - Перераспределить чанки по новым поддиапазонам.


#### 2.2. Автомасштабирование числа чанков
