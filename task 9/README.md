
 `products`

| Операция чтения | Источник (primary/secondary) | Допустимая задержка репликации | Обоснование |
|---------------|-----------------------------|-----------------------------|----------|
| Поиск товаров по категории и цене | **secondary** | до 30 сек | Данные обновляются нечасто (остатки при покупке), небольшая задержка не критична для UX. Пользователь видит актуальный ассортимент с минимальной погрешностью. |
| Просмотр карточки товара | **primary** | 0 сек | Требуется точная информация об остатке на складе *в момент просмотра*. Иначе риск показать доступный товар, который уже распродан. |
| Проверка остатка перед добавлением в корзину | **primary** | 0 сек | Критично для бизнес‑логики: нельзя допустить добавление в корзину товара с нулевым остатком. |


`orders`

| Операция чтения | Источник (primary/secondary) | Допустимая задержка репликации | Обоснование |
|---------------|-----------------------------|-----------------------------|----------|
| Список истории заказов пользователя | **secondary** | до 60 сек | История заказов редко обновляется. Задержка в минуту не влияет на восприятие: пользователь видит почти актуальные данные. |
| Текущий статус конкретного заказа | **primary** | 0 сек | Пользователь ожидает мгновенного обновления статуса (например, «отправлен»). Задержка может вызвать недоверие и повторные обращения в поддержку. |
| Сумма заказов за период (аналитика) | **secondary** | до 5 мин | Для агрегированной аналитики допустима существенная задержка: точность в реальном времени не требуется. |
| Проверка наличия заказа перед оплатой | **primary** | 0 сек | Необходимо гарантировать, что заказ не был отменён/изменён в последний момент. |

`carts`

| Операция чтения | Источник (primary/secondary) | Допустимая задержка репликации | Обоснование |
|---------------|-----------------------------|-----------------------------|----------|
| Получение активной корзины пользователя | **primary** | 0 сек | Корзина часто обновляется (добавление/удаление товаров). Задержка приведёт к отображению устаревшего состава и суммы. |
| Просмотр гостевой корзины по `session_id` | **primary** | 0 сек | Аналогично: изменения в корзине должны отображаться мгновенно. |
| Статистика количества активных корзин (мониторинг) | **secondary** | до 1 мин | Для операционных метрик допустима небольшая задержка: важна общая динамика, а не точное число в моменте. |
| Анализ abandonment rate (доля брошенных корзин) | **secondary** | до 15 мин | Аналитические отчёты не требуют реального времени. Задержка в 15 мин не влияет на выводы. |
