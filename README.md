# «Мобильный мир» — Sprint 4, проектная работа

Решение проектной работы по курсу «Архитектура программного обеспечения» (Яндекс.Практикум, спринт 4). Кейс — интернет-магазин «Мобильный мир», у которого в «Чёрную пятницу» легла MongoDB. Задача — переделать слой хранения, дать сайту шардирование, репликацию, кеш, и подготовить план дальнейшего роста.

## Как запустить финальный стенд (для ревью)

```shell
cd sharding-repl-cache
docker compose up -d
./scripts/init-sharding.sh
```

Скрипт инициализирует replica-сеты, добавляет шарды через `mongos`, шардирует `somedb.helloDoc` по `{ name: "hashed" }`, заливает 1000 документов и выполняет три подряд запроса к `/helloDoc/users` с замером времени, чтобы продемонстрировать работу Redis-кеша.

После завершения скрипта:

- Веб-интерфейс приложения — http://localhost:8080
- Swagger — http://localhost:8080/docs
- В корневом JSON-ответе будут видны:
  - `mongo_topology_type: "Sharded"` — топология шардированная;
  - `shards` — два реплика-сета `shard1` и `shard2`, каждый с тремя узлами;
  - `collections.helloDoc.documents_count: 1000` — данные залиты;
  - `cache_enabled: true` — Redis подключён.

Полная инструкция по запуску, проверке распределения данных и работе кеша — в [`sharding-repl-cache/README.md`](sharding-repl-cache/README.md).

### Требования к окружению

- Docker и Docker Compose (минимум 2 CPU, 4 GB RAM, как указано в задании).
- Доступ к Docker Hub и `dh-mirror.gitverse.ru` (используются образы `kazhem/pymongo_api:1.0.0` и `dh-mirror.gitverse.ru/mongo:latest` / `dh-mirror.gitverse.ru/redis:latest`).
- Свободные порты на хосте: `8080` (приложение), `6379` (Redis).

### Остановка

```shell
cd sharding-repl-cache
docker compose down -v
```

## Архитектурный документ (задания 7–10)

[`ARCHITECTURE.md`](ARCHITECTURE.md) содержит:

- **Задание 7** — спроектированные схемы коллекций `products`, `orders`, `carts` с выбором шард-ключей и обоснованием стратегий шардирования.
- **Задание 8** — набор метрик мониторинга шардов и стратегии устранения «горячих» шардов (balancer, refine shard key, resharding, zone sharding, архитектурные меры для категории «Электроника»).
- **Задание 9** — таблица операций чтения с указанием `readPreference` (primary / secondary) и `readConcern` для каждой коллекции с обоснованием через бизнес-логику.
- **Задание 10** — миграция на Cassandra: что переносить и почему, концептуальная модель с partition / clustering keys, стратегии целостности (Hinted Handoff, Read Repair, Anti-Entropy Repair) с компромиссами latency vs гарантии.
